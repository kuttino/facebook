git è un DVCS cioè un distribuited      version control system


una repository è una struttura dati (cioè un modo di raccogliere delle informazioni) che tiene traccia dei cambiamenti
apportati al progetto nel tempo

una repository è formata da uno o più branch

un branch è una linea di sviluppo (del progetto) nel tempo

un commit è una istantanea del progetto


git remote permette di stabilire la locazione della repository remota corrispondente alla repository locale
